# Mobile Terminal Verbetervoorstellen

De hero terminal heeft layout problemen op mobile devices. Hieronder zie je verschillende oplossingsrichtingen met live voorbeelden.

---

## Huidige Problemen

1. ❌ Tekst loopt buiten het terminal venster
2. ❌ Progress bar te breed voor small screens
3. ❌ Lange client namen wrappen niet
4. ❌ Font size (14px) is te klein op mobile
5. ❌ Padding (24px) neemt te veel ruimte

---

### CSS Wijzigingen

```css
/* terminal-window.module.css */
.terminal {
  @media (--md-n-below) {
    padding: var(--spacing-4); /* 16px instead of 24px */
  }
}

.content {
  word-wrap: break-word;
  overflow-wrap: break-word;
}

/* hero.module.css */
.terminal-window {
  @media (--md-n-below) {
    font-size: 12px; /* Smaller on mobile */
  }
}

.command,
.success {
  word-wrap: break-word;
  overflow-wrap: break-word;
}

.progress-bar {
  @media (--md-n-below) {
    max-width: 100%;
  }
}

.progress-label {
  @media (--md-n-below) {
    font-size: 10px;
  }
}
```

**Voordelen:**
- ✅ Alle content blijft zichtbaar
- ✅ Authentieke terminal look
- ✅ Minimale code wijzigingen
- ✅ Geen content verlies

**Nadelen:**
- ⚠️ Tekst wrap kan "onecht" aanvoelen voor een terminal
